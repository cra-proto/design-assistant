About AIDA

The DDPD GenAI Project is exploring how artificial intelligence and automation can improve web content design for Canada.ca.

The AI Design Assistant (AIDA) helps content designers manage projects, analyze pages, identify issues, and create prototypes.

What it does now

  • Generate and manage your content inventory
  • Visualize your information architecture
  • Update your metadata with AI assistance
  • Export your prototypes to GitHub for editing

What's coming next

  • AI-assisted content review
  • Accessibility checking
  • Automated quality assurance tools
  • Have an idea? Contact CDIA.

How AIDA collects its data

When you add a Canada.ca page to your project, AIDA fetches information from the live English and French pages, the live content.json file, the task airtable, and from a static export of page visists from the UPD.

Page templates

AIDA detects page templates based on specific criteria and puts that information in the "Template" column of your content inventory table. This helps give you an overview of what kind of pages you're working with. Here's what AIDA looks for:

Navigation templates:

  • topic - has classes most-requested-bullets or gc-srvinfo or mwsdoormat-links-container
  • old topic - has class list-group-item where more than 80% are links
  • subway - has class gc-subway
  • old subway - has class gc-navseq
  • navigation - where none of the above apply and more than 70% of the text is links

Content templates

  • multimedia gallery - url includes /news/cra-multimedia-library/ or /nouvelles/bibliotheque-multimedia-arc/
  • video transcript - has <video> element, H2 contains "transcript", and is a child of the multimedia gallery
  • contact - h1 starts with "Contact" or "Contactez"
  • brochure - has classes panel-heading bg-primary
  • pdf download - has a pdf link with classes btn stretched-link and either the thumbnail class or a <small> element with "PDF, # KB, # page" or "PDF, # Ko, # page"
  • content - if no other page templates apply

Newsroom templates:

  • newsroom - url includes /news/#### or /nouvelles/####
  • taxtip - url includes /newsroom/tax-tips/ or /salle-presse/conseils-fiscaux/
  • tax filing season media kit - url includes /tax-tips/tax-filing-season-media-kit or /salle-presse/mesures-relatives-enquetes-criminelles-accusations-condamnations
  • enforcement notice - url includes /newsroom/criminal-investigations-actions-charges-convictions or /salle-presse/mesures-relatives-enquetes-criminelles-accusations-condamnations
  • campaign - url includes /campaigns/ or /campagnes/

Forms and publications templates:

  • readme (form) - url includes /forms-publications/forms/ or /formulaires-publications/formulaires/
  • readme (guide) - url includes /forms-publications/publications/[guide number].html or /formulaires-publications/publications/[guide number].html
  • readme (T1) - url includes /general-income-tax-benefit-package/[province]?/5###-[1 to 5 letters].html or /trousse-generale-impot-prestations/[province]?/5###-[1 to 5 letters].html
  • readme (TD1) - url includes /td1-forms-pay-received-on-january-1-[year]?-later/[filename].html or /formulaires-td1-paies-recues-1er-janvier-[year]?-apres/[filename].html
  • readme (payroll) - url includes:
    • /t4127-payroll-deductions-formulas/ or /t4127-formules-calcul-retenues-paie/
    • /payroll-deductions-t4127-payroll-deductions-formulas/ or /t4127-formules-calcul-retenues-paie-annees-precedentes/
    • /t4032-payroll-deductions-tables/ or /t4032-tables-retenues-paie/
    • /t4032-payroll-deductions-tables-previous-years/ or /t4032-tables-retenues-paie-documents-annees-anterieures/
    • /t4008-payroll-deductions-supplementary-tables/ or /t4008-tables-supplementaires-retenues-paie/
    • /t4008-payroll-deductions-supplementary-tables-previous-years/ or /t4008-tables-supplementaires-retenues-paie-annees-anterieures/
    • followed by [filename].html
  • guide - url includes /forms-publications/publications/ or /formulaires-publications/publications/ if readme (guide) doesn't apply
  • guide (T1) - url includes:
    • /general-income-tax-benefit-package/5000-g.html or /trousse-generale-impot-prestations/5000-g.html
    • /general-income-tax-benefit-package/[province]?/5###-[1 to 5 letters]/[guide].html or /trousse-generale-impot-prestations/[province]?/5###-[1 to 5 letters]/[guide].html

If AIDA tags a page incorrectly or if you think we're missing a template type, please let us know.

Release notes

Release 0.5.3 (March 31, 2026)

GitHub export improvements

  • Add option to export English, French or both languages for prototyping

Update bookmarklets

  • Update "Toggle [repo name] and "Toggle github" to work with new cra-test-arc.canada.ca subdomain

Update template detection

  • Add "mwsdoormat-links-container" to topic page template detection logic

Release 0.5.2 (March 27, 2026)

Add pages improvements

  • Added input validation to enforce single-language page entry
  • Automatic URL formatting and normalization
  • Removed list of urls undergoing validation (we still show urls with problems and valid or skipped urls)
  • Added "View pages" section to the Edit/New project view

Github export improvements

  • Improved GitHub index.html to support bilingual sitemap generation with paired English/French URLs

Release 0.5.1 (March 26, 2026)

New bookmarklet

  • Added project-specific "Toggle [repo name]" bookmarklet to toggle between GitHub preview, edit mode, and Canada.ca.

Release 0.5.0 (March 25, 2026)

AI-powered metadata generation

  • Generate metadata automatically for selected pages using AI assistance

Content inventory improvements

  • Expanded data collection: word count, noindex status, last modified/published dates
  • Expanded metadata collection to both languages
  • Secondary toolbar for bulk actions on selected pages
  • Improved page status filtering
  • Added option to refresh the collected page data
  • Reorganized IA orphan detection into dedicated problem category

Switch project improvements

  • Fixed cloud project upload with GitHub Personal Access Tokens
  • Filters on switch project view are now linked to the project files

GitHub export improvements

  • Added an index.html page to generate a list of files in your repo and robots.txt
  • Breadcrumbs now reflect any changes you've made in AIDA instead of the live page

New toolbox view

  • Added bookmarklets, "Open in AIDA", "Toggle GitHub", and "Check links"
  • Note: this view will eventually have some standalone tools that aren't linked to project data

Release 0.4.0 (February 25, 2026)

Initial release - core project management

  • Project dashboard view with high level stats and design phase tracking (discover, assess, design, approve)
  • Edit/New project view where you can name your project, associate it with a GitHub repo, add collaborators, add pages, or find pages from the task inventory
  • Switch project view where you can switch between your projects, view any cloud project, or mark projects for deletion (they are stored for 30 days before final deletion)
  • Manage inventory view where you can see all the data collected for your pages, add brand new pages, or change the IA structure
  • IA diagram view where you can see the page hierarchy
  • Export to GitHub view where you can export your pages in Jekyll format. The file comparison table can detect if you've made changes in GitHub after the export and will automatically skip those files on future exports (can be overriden by the user).
